=== Общая схема архитектуры

[cols="1^8"]
|===

|Компонент|Описание

|Клиенты (UI)|Пользователи - сотрудники предприятия использующие для доступа к сервису - личные мобильные устройства на IOS и Android, десктопные и веб версии мессенджера

|API Gateway|Входная точка для подключения клиентских мессенждеров, их авторизации. Используется HTTPS и websocket. Маршрутизирует запросы от пользователя между компонентами системы. Реализует rate limiting, circuit breaker, retry.

|Load Balancer|Распределяет входящий трафик между репликами API Gateway. Обеспечивает балансировку нагрузки и отказоустойчивость.

|Service Discovery|Динамический поиск сервисов и их регистрация. Использует Kubernetes DNS или Consul для обнаружения микросервисов.

|Сервис авторизации|Аутентифицирует пользователя - на основании информации из сервиса пользователей, предоставляет авторизацию на различные действия например блокировку пользователей - глобальное удаление сообщений. Также сервис имплементирует one time password чтобы не смешивать общий пароль пользователя на предприятии и пароль для мессенждера.

|Сервис директ чатов|Реализует функциональность директ чатов пользователь - пользователь - хранит данные чатов в БД PostgreSQL. Использует CQRS для разделения чтения и записи.

|Сервис пользователей|Хранит информацию о пользователях (postgreSQL), полученную от внешней интеграции с LDAP предприятия, так и информацию которую пользователь указал о себе, статус блокировки пользователя (например в случае увольнения) аватары и т.д.

|Сервис медиа|Хранит и отдает мультимедиа материалы (картинки, документы, видео) хранящиеся в s3 хранилище. метаинформация о медиа (пользователь - владелец, ссылка на s3) хранится в postgreSQL. Использует CDN для снижения нагрузки при публикации медиа в большие каналы.

|Сервис групповых чатов|Реализует функциональность каналов и групповых чатов - использует postgreSQL. Поддерживает треды, упоминания, хэштеги.

|Сервис уведомлений|Реализует функциональность доставки пользователям сообщений из групповых и директ чатов онлайн и после входа пользователя после отутствия - использует postgreSQL. Также используется для взаимодействия с mail сервером для пересылки OTP пользователю.

|Брокер сообщений (Kafka)|Kafka для асинхронного взаимодейтствия между сервисами. Используется для событий: MessageCreated, MediaUploaded, UserBlocked, NotificationRequired.

|Сервис поиска|Поиск пользователем по истории директ и групповых чатов и каналов Использует Elasticsearch. Индексирует сообщения в реальном времени.

|Сервис безопасности|Реализует E2EE (end-to-end encryption) с использованием Signal Protocol. Управляет ключами шифрования.

|Базы данных|PostgreSQL (реляционные данные), Redis (кэш, сессии), Elasticsearch (поиск), S3/MinIO (медиа).

|Мониторинг и логирование|Prometheus (метрики), Grafana (дашборды), ELK Stack (логи)

|===

=== Схема взаимодействия компонентов

[cols="2^8"]
|===

|Уровень|Компоненты|Протоколы

|Периметр|Load Balancer ->  API Gateway|HTTPS, HTTP/2

|Граничный уровень|API Gateway -> Service Discovery|gRPC, DNS

|Сервисы|API Gateway -> Auth Service, User Service, Chat Service, Media Service, Notification Service, Search Service|gRPC (Protobuf), REST (JSON)

|Брокер сообщений|Chat Service, Media Service, Notification Service -> Kafka|TCP (Kafka Protocol), Avro/JSON

|Хранилища|Chat Service, User Service -> PostgreSQL|TCP (PostgreSQL Protocol)
|Media Service -> S3/MinIO|HTTPS (REST API)
|Search Service -> Elasticsearch|HTTPS (REST API)
|Все сервисы -> Redis|TCP (Redis Protocol)

|Внешние интеграции|Auth Service -> LDAP|LDAP + TLS
|Notification Service -> Mail Server|SMTP + TLS


|===

=== Потоки данных

[cols="2^8"]
|===

|Поток|Направление|Протокол|Критичность

|Авторизация|Клиент -> API Gateway -> Auth Service -> User Service -> LDAP -> Клиент|HTTPS, gRPC, LDAP|Высокая

|Отправка сообщения|Клиент -> API Gateway -> Chat Service -> PostgreSQL -> Kafka -> Notification Service -> Push Provider -> Клиент|HTTPS, gRPC, TCP, HTTPS|Высокая

|Загрузка медиа|Клиент -> API Gateway -> Media Service -> S3 -> Kafka -> Search Service|HTTPS, HTTPS, TCP|Средняя

|Поиск|Клиент -> API Gateway -> Search Service -> Elasticsearch -> Клиент|HTTPS, HTTPS|Средняя

|Уведомления|Chat Service -> Kafka -> Notification Service -> Клиент|TCP, HTTPS|Высокая

|===

=== Масштабируемость компонентов

[cols="2^8"]
|===

|Компонент|Масштабируемость|Механизм

|API Gateway|Горизонтальное масштабирование|Добавление реплик, балансировка нагрузки

|Load Balancer|Горизонтальное масштабирование|Добавление реплик, DNS round-robin

|WAF|Горизонтальное масштабирование|Кластеризация, распределение правил

|Сервис авторизации|Горизонтальное масштабирование + кэширование|Реплики, Redis для JWT

|Сервис чатов|CQRS + горизонтальное масштабирование|Разделение чтения/записи, реплики PostgreSQL

|Сервис медиа|Горизонтальное масштабирование + CDN|Реплики, CDN для доставки

|Сервис уведомлений|Горизонтальное масштабирование + Kafka|Реплики, Kafka партиции

|Сервис поиска|Кластеризация + шардинг|Elasticsearch ноды, шарды

|Kafka|Кластеризация|Брокеры, партиции, репликация

|PostgreSQL|Репликация + шардинг|Primary-replica, шардинг по user_id

|Redis|Кластеризация|Redis Cluster, репликация

|Elasticsearch|Кластеризация + шардинг|Ноды, шарды, реплики

|===
