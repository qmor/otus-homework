=== Аудит безопасности

==== Логирование событий безопасности

===== Типы событий

**Аутентификация:**
- Успешный вход
- Неудачный вход
- Выход из системы
- Просроченный токен

**Авторизация:**
- Успешная проверка прав
- Ошибка авторизации
- Попытка доступа к запрещённому ресурсу

**Доступ к данным:**
- Чтение пользовательских данных
- Чтение сообщений
- Экспорт данных

**Управление доступом:**
- Создание/изменение/удаление роли
- Назначение роли пользователю

**Управление секретами:**
- Чтение/запись секрета
- Ротация секрета
- Ошибка доступа к секрету



===== Хранение логов

**Инструменты:**
- Elasticsearch - хранение и поиск
Сбор логов через stdout сервиса в kubernetes


**Политика хранения:**
- Горячие данные: 7 дней
- Тёплые данные: 30 дней
- Холодные данные: 90 дней
- Удаление: после 90 дней

==== Ключевые метрики

===== Аутентификация

1. **Количество успешных входов** - Counter
2. **Количество неудачных входов** - Counter
3. **Коэффициент неудачных входов** - Gauge (тревога: > 10%)
4. **Время ответа на аутентификацию** - Histogram (тревога: p95 > 1s)

===== Авторизация

5. **Количество отказов в доступе** - Counter
6. **Коэффициент отказов** - Gauge (тревога: > 5%)

===== Секреты

7. **Количество чтений секретов** - Counter
8. **Количество ошибок доступа** - Counter

===== Аномальное поведение

9. **Запросы с одного IP** - Gauge (тревога: > 100/мин)
10. **Запросы от одного пользователя** - Gauge (тревога: > 50/мин)
11. **Неудачные попытки с одного IP** - Gauge (тревога: > 5 за 5 мин)

==== Обнаружение аномалий

===== Сценарии атак

**1. Brute Force Attack**
- Признаки: много неудачных попыток с одного IP
- Действия: блокировка IP на 15 минут

**2. Credential Stuffing**
- Признаки: много успешных входов с разных IP
- Действия: требование 2FA, уведомление пользователя

**3. Data Exfiltration**
- Признаки: аномально много запросов к API
- Действия: блокировка пользователя, уведомление администратора

**4. Unusual Geographic Activity**
- Признаки: вход с необычной страны
- Действия: требование дополнительной верификации


==== Аудит безопасности

===== Периодические проверки

**Еженедельный:**
- Проверка логов безопасности
- Анализ аномальной активности
- Проверка ротации секретов

**Ежемесячный:**
- Проверка прав доступа пользователей
- Анализ уязвимостей
- Обновление политик безопасности

**Квартальный:**
- Полный пересмотр архитектуры безопасности
- Тестирование на проникновение
- Обучение сотрудников

===== Отчёты

**Еженедельный:**
- Количество событий безопасности
- Количество обнаруженных атак
- Статус инцидентов

**Ежемесячный:**
- Обзор метрик безопасности
- Анализ трендов
- План действий

**Квартальный:**
- Обзор архитектуры безопасности
- Результаты тестирования
- План улучшений

==== Реагирование на инциденты

===== Процедура

1. **Обнаружение** - через алерты или дашборды
2. **Классификация** - тип и уровень серьёзности
3. **Сдерживание** - блокировка IP, отключение аккаунтов
4. **Устранение** - устранение уязвимости
5. **Восстановление** - восстановление из бэкапов
6. **Анализ** - анализ причин, обновление процедур

===== Playbooks

**Brute Force Attack:**
1. Блокировать атакующий IP
2. Уведомить администраторов
3. Включить 2FA или капчу
4. Мониторить ситуацию

**Data Exfiltration:**
1. Блокировать скомпроментированный аккаунт
2. Уведомить администраторов
3. Анализировать экспортированные данные
4. Восстановить из бэкапов

**Unusual Geographic Activity:**
1. Включить 2FA
2. Уведомить пользователя
3. Временная блокировка аккаунта
